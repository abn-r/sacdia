Cambios de schema propuestos para honor_requirements

Campos nuevos necesarios para soportar sub-items, evidencia, y choice groups

Schema actual

// honor_requirements (actual)
requirement_id Int PK autoincrement
honor_id Int FK → honors
requirement_number Int ordenamiento
requirement_text String texto completo (con sub-items embebidos)
has_sub_items Bool flag simple
needs_review Bool pendiente de revisión
active Bool
created_at/modified_at DateTime
@@unique([honor_id, requirement_number])

Schema propuesto

// honor_requirements (propuesto)
requirement_id Int PK autoincrement (sin cambios)
honor_id Int FK → honors (sin cambios)
parent_id Int? NUEVO — FK → honor_requirements (null = top-level)
requirement_number Int ordenamiento (sin cambios)
display_label String? NUEVO — "1", "a", "ii", "3.b" (para mostrar al usuario)
requirement_text String solo texto instruccional (sin sub-items embebidos)
reference_text String? NUEVO — tablas/material de referencia separado
has_sub_items Bool (sin cambios, derivado de children.count > 0)
is_choice_group Bool NUEVO — true para "elegí N de los siguientes"
choice_min Int? NUEVO — mínimo a completar (N en "elegí N de M")
requires_evidence Bool NUEVO — true si es práctico (fogata, construir, etc.)
needs_review Bool (sin cambios)
active Bool
created_at/modified_at DateTime
@@unique([honor_id, requirement_number, parent_id])

Tabla nueva: requirement_evidence

// requirement_evidence (NUEVA)
evidence_id Int PK autoincrement
progress_id Int FK → user_honor_requirement_progress
evidence_type Enum IMAGE | FILE | LINK
url String URL del archivo (R2) o enlace externo
filename String? nombre original del archivo
mime_type String? tipo MIME para archivos
file_size Int? tamaño en bytes
active Bool
created_at/modified_at DateTime
@@index([progress_id])
// Límite: 3 por tipo por progress_id (enforced en app layer)
