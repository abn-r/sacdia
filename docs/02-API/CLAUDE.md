## 5) Convenciones clave

- Naming: `camelCase` en TS y `snake_case` en SQL.
- Versionado de endpoints: `URI-based`, usar `/api/v1/...`.
- Validacion de input: DTOs con `class-validator` + `class-transformer`.
- Manejo de errores: errores tipados + respuestas consistentes, no exponer detalles internos.
- Auth/RBAC: Supabase JWT + `SupabaseGuard` + `RolesGuard` + permisos por dominio.
- Seguridad: Helmet, Throttler, CORS estricto, sanitizacion de input.

## 6) Limites y riesgos

- No tocar sin analisis previo: flujo de auth, guards globales, middlewares de seguridad, auditoria.
- Riesgo alto: cambios en contratos de endpoints ya consumidos por `sacdia-admin` y `sacdia-app`.
- Riesgo medio: cambios en DTOs, filtros/paginacion, codigos de error.
- Dependencias externas criticas: Supabase, Redis (Upstash), Firebase FCM, Sentry.
